# Hubbly Frontend

Современный frontend для социальной платформы Hubbly, построенный на React + TypeScript + Vite.

## 🚀 Быстрый старт

### Установка зависимостей

```bash
npm install
```

### Настройка окружения

Создайте файл `.env` в корне проекта:

```env
VITE_API_URL=http://localhost:8000
```

### Запуск разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:3000`

### Сборка для production

```bash
npm run build
```

## 📁 Структура проекта

```
frontend/
├── src/
│   ├── app/              # Настройка приложения (router, providers)
│   ├── shared/           # Переиспользуемые компоненты и утилиты
│   │   ├── api/          # API клиент и interceptors
│   │   ├── components/   # UI компоненты (shadcn/ui)
│   │   ├── hooks/        # Переиспользуемые хуки
│   │   ├── lib/          # Утилиты
│   │   └── types/        # TypeScript типы
│   ├── features/         # Feature-модули
│   │   ├── auth/         # Авторизация
│   │   ├── users/        # Пользователи
│   │   ├── posts/        # Посты
│   │   ├── events/       # События
│   │   ├── communities/  # Сообщества
│   │   └── notifications/# Уведомления
│   └── widgets/          # Составные компоненты
└── public/               # Статические файлы
```

## 🏗️ Архитектура

Проект использует **Feature-Sliced Design** архитектуру:

- **app/** - Инициализация приложения
- **shared/** - Переиспользуемый код
- **features/** - Бизнес-логика по доменам
- **widgets/** - Композиция features

## 🛠️ Технологии

- **React 18** - UI библиотека
- **TypeScript** - Типизация
- **Vite** - Сборщик
- **React Router** - Маршрутизация
- **React Query** - Управление состоянием сервера
- **Axios** - HTTP клиент
- **Tailwind CSS** - Стилизация
- **shadcn/ui** - UI компоненты
- **React Hook Form + Zod** - Формы и валидация

## 📝 API Integration

Все API вызовы настроены через axios с автоматическим:
- Добавлением JWT токенов
- Обновлением токенов при истечении
- Обработкой ошибок
- Базовым URL из `.env`

## 🔐 Авторизация

Приложение использует JWT токены:
- Access token в заголовках запросов
- Refresh token для обновления access token
- Автоматический logout при ошибках авторизации
